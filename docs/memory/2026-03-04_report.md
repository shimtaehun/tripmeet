### [작업 5] 흰 화면 버그 수정 (`cf559e1`)

**원인 분석:**

`LandingScreen.tsx`에서 `View` ref에 `appendChild`를 직접 호출하는 방식이 불안정:
```tsx
// 기존 방식 (문제)
const container = containerRef.current;
if (container) {
  container.appendChild(iframe);  // View ref가 DOM element를 항상 노출하지 않음
}

// 수정 후 방식
const Iframe = 'iframe' as any;
return (
  <View style={{ flex: 1 }}>
    <Iframe
      srcDoc={LANDING_HTML}
      style={{ width: '100%', height: '100%', border: 'none', display: 'block' }}
      title="TripMeet Landing"
    />
  </View>
);
```

React Native Web에서 HTML 요소(`iframe`)를 `'iframe' as any` 타입 캐스팅으로 JSX에서 직접 렌더링하는 방식으로 교체.

**package.json build 스크립트 추가:**
```json
"build": "expo export --platform web"
```
Render.com Build Command: `npm install && npm run build` 로 단순화.

### [작업 6] 모바일 네비게이션 바 반응형 수정 (`8dcf991`)

**문제:** 모바일(900px 이하)에서 로고 + 메뉴링크 3개 + CTA 버튼이 한 줄에 들어가지 않아 레이아웃 깨짐

**수정:**
```css
@media (max-width: 900px) {
  nav { padding: 14px 20px; }
  .nav-links { display: none; }        /* 메뉴 링크 숨김 */
  .nav-cta { padding: 10px 18px; font-size: 13px; }
}
```

`landing/index.html`과 `LandingScreen.tsx` LANDING_HTML 양쪽 동일 적용.
